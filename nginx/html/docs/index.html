An Library Management Web Application

Created By: Partha Pratim Samanta

Summary of this project:

Scenario

The are two roles in the system; LIBRARIAN and MEMBER

As a User

  • I can signup either as LIBRARIAN and MEMBER using username and password
  • I can login using username/password

As a Librarian

  • I can add, update, and remove Books from the system
  • I can add, update, view, and remove Member from the system

As a Member

  • I can view, borrow, and return available Books
  • Once a book is borrowed, its status will change to BORROWED
  • Once a book is returned, its status will change to AVAILABLE
  • I can delete my own account

    Testing Process

Librarian:

  • Sign UP as Librarian (Staff)
    • Method: POST
    • Sample POST Data: 
        {
    "name": "Your Name/username",
    "password": "Your password"
  }
  • Sign In as Librarian (Staff)
    • Method: POST
    • Content (Credential): 
        {
    "name": "Your Name/username",
    "password": "Your password"
  }

Books:

  • See list of all available books

    • Login Required: Staff/Member
    • Method: GET

  • Add a new book

    • Login Required: Staff
    • Method: POST 
      {
"name": "Name of the Book!"
}

  • Update an existing book

    • Login Required: Staff
    • Method: POST
    • replace <book_id> with a valid integer in the URL
    • Sample POST Data: 
        {
    "name": "Name of the Book!",
    "status": "BORROWED",
    "borrower": 102
  }
    • borrower must be a valid member id.

  • Delete a book

    • Login Required: Staff
    • Method: POST
    • replace <book_id> with a valid integer in the URL

Member:

  • Sign Up as a Member

    • Method: POST
    • Sample POST Data: 
        {
    "name": "Name of  the Member",
    "password": "Password of the Member"
  }

  • Log In as a Member

    • Method: POST
    • Sample POST Data: 
        {
    "name": "Name of  the Member",
    "password": "Password of the Member"
  }

  • View list of all members

    • Login Required: Staff
    • Method: GET

  • Add a new member

    • Login Required: Staff
    • Method: POST
    • Sample POST Data: 
        {
    "name": "Member Ten",
    "password": "memberten"
  }

  • Update Member
          Login Required: Staff
          Method: POST

    • replace <member_id> with a valid integer in the URL
    • Sample POST Data: 
        {
    "name": "Member Ten Updated"
    "password": "membertenupdt"
  }

  • Delete Member
          Login Required: Staff/Member
          Method: POST

    • replace <member_id> with a valid integer in the URL
    • if login as staff: can delete any member.
    • if login as member: can delete only his account.

  • Borrow a book
          Login Required: Member
          Method: POST

    • can't be borrowed if already done
    • replace <book_id> with a valid integer in the URL

  • Return a book
          Login Required: Member
          Method: POST

    • can't be returned if not borrowed or borrowed by someine else.
    • replace <book_id> with a valid integer in the URL
